# 🌐 network-tools — 本地网络检测工具

一个用 Rust 编写的轻量级命令行工具，用于快速诊断本机网络状态。

## 功能

| 模块 | 说明 |
|------|------|
| 📡 **网络接口列表** | 列出所有网卡的名称、MAC、IPv4、状态、类型（物理/虚拟/VPN/TUN）、接口跃点、是否为出口 |
| 🚪 **流量出口** | 通过 UDP 探测识别实际出口接口，并展示选路逻辑 |
| 🗺️ **路由表** | 默认路由优先展示，包含目标网段、网关、接口、跃点数 |
| 🔒 **代理设置** | 检测环境变量代理（HTTP_PROXY 等）和 Windows 系统代理（注册表） |

## 快速开始

```bash
# 编译
cargo build --release

# 运行
cargo run --release

# 或直接运行编译产物
./target/release/network-tools.exe
```

## 依赖

| 依赖 | 用途 |
|------|------|
| `winreg` | 读取 Windows 注册表中的系统代理设置 |

> 网络接口、路由表等信息通过 PowerShell 命令获取，无需额外依赖。

## 出口标记说明

| 标记 | 含义 |
|------|------|
| `✓ 出口` | UDP 探测到的**实际出口接口**（系统真正选中的） |
| `~ 备用` | 路由表中有默认路由，但不是实际出口 |

## 系统选路原理

Windows 选择出口接口时按以下顺序决策：

1. **最长前缀匹配**：目标 IP 先匹配最精确的路由条目
2. **有效跃点比较**：`有效跃点 = RouteMetric + InterfaceMetric`，越低越优先
3. **接口优先级**：跃点相同时按接口绑定顺序决定

TUN/VPN 工具（如 Mihomo、Clash）通常在启动时将自己的 `InterfaceMetric` 设为 0，确保流量优先走虚拟网卡。

## 接口跃点参考

| 接口类型 | 典型跃点 |
|----------|----------|
| TUN/VPN 虚拟网卡 | 0 |
| TAP 虚拟网卡 | 1 |
| 物理以太网 | 25 |
| 无线网卡 | 25-50 |
| Loopback 回环 | 75 |
| Hyper-V/WSL | 5000 |

## 支持的虚拟网卡类型

工具能自动识别以下虚拟网卡类型：

- **Mihomo/TUN** — Mihomo (Clash Meta) TUN 模式
- **Clash/TUN** — Clash TUN 模式
- **OpenVPN** — OpenVPN 客户端
- **WireGuard** — WireGuard VPN
- **VirtualBox** — VirtualBox Host-Only 适配器
- **VMware** — VMware 虚拟网卡
- **Hyper-V** — Hyper-V / WSL 虚拟交换机
- **Docker** — Docker 网桥
- **TUN/TAP** — 通用 TUN/TAP 设备
- **Radmin VPN / ZeroTier / Tailscale** — 常见组网工具
